Email Verification Route (Signed URL Handler)
routes/web.php Route
Route::get('/email/verify/{id}/{hash}', function($id, $hash) { $user = User::findOrFail($id); if (! hash_equals((string) $hash, sha1($user->getEmailForVerification()))) abort(403); if ($user->markEmailAsVerified()) { event(new Verified($user)); } return redirect()->route('login')->with('success', 'Email verified!'); })->middleware(['signed', 'throttle:6,1'])->name('verification.verify');

Model Scope Search (10pts level)
app/Models/Product.php Model Scope
scopeSearch() — Reusable Query Scope
public function scopeSearch($query, $search) { return $query ->where('name', 'like', "%{$search}%") ->orWhere('series', 'like', "%{$search}%") ->orWhere('brand', 'like', "%{$search}%") ->orWhere('description', 'like', "%{$search}%"); } // Usage: Product::search('hirono')->get()
Local Query Scope — methods prefixed with scope become chainable methods on the model (called without the prefix: ->search()). This encapsulates the LIKE search logic in the model, making it reusable across controllers. This is the "model search" implementation versus inline LIKE queries.
LIKE Search (8pts level — ProductController)
app/Http/Controllers/ProductController.php Controller LIKE Query
$query->where(function($q) use ($s) { $q->where('name', 'like', "%{$s}%") ->orWhere('series', 'like', "%{$s}%") ->orWhere('brand', 'like', "%{$s}%") ->orWhere('description', 'like', "%{$s}%"); });
Basic SQL LIKE search. The closure groups the OR conditions correctly — without the wrapping closure, additional where() calls after the OR block would be incorrectly joined, producing wrong results. The % wildcards match any characters before or after the search term.

Checkout — DB Transaction + Confirmation Email

store() — Database Transaction
DB::beginTransaction(); try { // 1. Check stock foreach ($cartItems as $item) { if ($item->product->stock_quantity < $item->quantity) throw new Exception('Insufficient stock...'); } // 2. Create order $order = Order::create([...]); // 3. Create order items + decrement stock foreach ($cartItems as $item) { OrderItem::create([...]); $item->product->decrement('stock_quantity', $item->quantity); } auth()->user()->cartItems()->delete(); DB::commit(); } catch (Exception $e) { DB::rollBack(); return back()->with('error', $e->getMessage()); }
DB::beginTransaction() — wraps multiple database operations in an atomic unit. If any step fails (stock check, order creation, item creation), rollBack() undoes all changes, ensuring data integrity. Only on success does commit() finalize all changes. decrement() atomically reduces stock count. This prevents overselling.
Email After Transaction
Mail::to($order->user->email)->send( new OrderReceiptMail($order->fresh(['orderItems.product', 'user'])) );

2nd Normal Form Compliance
Schema Design 2NF
2NF Explanation
The schema meets 2NF: every non-key attribute depends on the whole primary key. Key examples: product_photos — photos are separated from products (a product can have many photos; storing them inline would violate 1NF). order_items — quantity and unit_price are in a separate table because they depend on BOTH order_id AND product_id, not just the order. reviews — linked to both user_id and product_id and order_id, ensuring no partial dependencies. All FK relationships are explicit and indexed.

Middleware Summary ALL MPs
MW How Middleware Protects the Entire Application
'guest' middleware — Login / Register routes
Built-in Laravel middleware. Redirects already-authenticated users away from the login/register pages. Applied with ->middleware('guest') on /login and /register.
'auth' middleware — Customer routes (cart, orders, profile, reviews)
Built-in. Redirects unauthenticated users to the login page. Applied to all routes that require being logged in.
'verified' middleware — All protected routes
Built-in. Works with the MustVerifyEmail interface. Redirects users with unverified email to the verification notice page. Ensures unverified users cannot use the shop.
'admin' middleware (EnsureUserIsAdmin) — All /admin/* routes
Custom middleware registered in bootstrap/app.php. Checks $user->role === 'admin'. Aborts with 403 if not admin. This is what separates the admin panel from the customer-facing site.
'throttle:6,1' middleware — Verification routes
Built-in rate limiting. Limits the email verification link to 6 attempts per 1 minute per IP. Prevents brute-force attacks against the verification endpoint.
'signed' middleware — Verification link
Built-in. Validates that the URL contains a valid cryptographic signature generated by Laravel. Prevents attackers from manually crafting verification URLs.
