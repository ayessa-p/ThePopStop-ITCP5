MP1 – Product / Service CRUD

1) Product/Service CRUD (create/read/update/delete products)

File: app/Http/Controllers/Admin/ProductController.php (lines ~26-54 for list, 60-81 for create, 88-106 for update, 108-113 for delete)
How it works:
  • List: Uses Yajra\DataTables to handle server-side processing for the product table in index() method.
  • Access Control: Protected by admin middleware in the constructor to restrict access to administrators only.
  • Add Product: store() method validates input via StoreProductRequest and saves new product records.
  • Update Product: update() method handles editing existing product details and managing image replacements.
  • Delete Product: destroy() method uses Soft Deletes to remove products while keeping them in the database for potential restoration.

2) Product/Service CRUDRestore (Soft Delete & Restore)

File: app/Http/Controllers/Admin/ProductController.php (lines ~46-52 for restore button, 115-121 for restore logic)
How it works:
  • Soft Deletes: The Product.php model uses the SoftDeletes trait.
  • Restore Action: The restore() method finds a trashed product by ID and calls $product->restore() to bring it back to the active list.

3) Product/Service CRUD - upload a single photo

File: app/Http/Controllers/Admin/ProductController.php (lines ~67-73 for store, 95-100 for update)
How it works:
  • On Add: Checks for $request->hasFile('image'), stores it in the public/products directory using Laravel's Storage facade, and saves the path to the image_url column.
  • On Update: If a new image is uploaded, it deletes the old file using Storage::disk('public')->delete() before saving the new one.

4) Product/Service CRUD - upload/view multiple photos

File: app/Http/Controllers/Admin/ProductPhotoController.php (entire file)
How it works:
  • Gallery Management: Each product can have multiple photos stored in the product_photos table.
  • Upload: store() method handles multiple file uploads, associating each with the product ID.
  • Primary Image: Allows users to set one of the gallery images as the 'Primary' photo which becomes the main thumbnail.

5) Product/Service CRUD - import products on excel worksheet

File: app/Http/Controllers/Admin/ProductController.php (lines ~123-133), app/Imports/ProductsImport.php
How it works:
  • Excel Processing: Uses Maatwebsite\Excel to parse uploaded .xlsx or .csv files.
  • Batch Import: ProductsImport class maps Excel columns to database fields and handles image downloading from external URLs provided in the sheet.

MP3 – Authentication & Security

1) Authentication and Email Verification
